What it is

One slash command runs a three-phase pipeline. Each phase uses the cheapest model that can do the job well, and the expensive tiers are reserved for the decisions that actually need them.

① Plan — OpusDecomposes the task into a JSON execution plan with a confidence score. Low confidence → Fable/Opus strengthens it. High risk → pauses for confirmation (unless [auto]).
↓
② Execute — Haiku + local OllamaSmart-selects the best installed Ollama model, generates each step locally, and the Haiku Worker adapts & writes. Oversized steps are chunked; Sonnet checks each chunked step.
↓
③ Review — Sonnet (→ Opus)Hard test gate: the suite must pass or the run is rejected. Escalates to Opus only when confidence is low. Failures trigger a cheap Sonnet-guided fix loop before any re-plan.
↓
④ MetricsLogs the run, checks the 170k token budget, and opens a local cost dashboard with real per-tier token usage and Ollama savings.

Primary features

Everything is designed around one goal: get production-quality output while keeping the expensive models out of routine work.

💸Cost-optimised routing

Opus only plans & reviews high-risk diffs; Haiku and local Ollama do the bulk implementation. Tier aliases auto-track the latest models.

🖥️Local Ollama offload

Three Worker modes: default (Haiku adapts an Ollama draft), [ollama-only] (verbatim write), and [ollama-agent] (the local model drives the step via its own tool-calling loop).

🌐OpenRouter backend

Prefer a hosted model over local? The [openrouter] flag routes Worker generation through OpenRouter (any model id) instead of Ollama — set OPENROUTER_API_KEY + OPENROUTER_MODEL. The two backends stay separate.

🎯Smart model selection

At probe time it scores your installed models for coding via llm-checker and picks the best one — no hardcoded model names. Pin one with [model:…].

📐Context-aware chunking

Detects each Ollama model's context window. If a step won't fit, Haiku splits it → Ollama generates the pieces → Haiku stitches → Sonnet verifies the stitched step.

✅Hard test gate

The reviewer must run the test suite. If any test fails, the run is forced to rejected — enforced at the orchestration level, never approved with failing tests.

🔧Tiered failure recovery

A test failure first triggers a cheap Sonnet-guided fix loop (Haiku/Ollama applies the fix) before escalating to a full Opus re-plan — fixing localized bugs without paying for a new plan.

📊Cost metrics dashboard

A local read-only dashboard compares actual mixed-tier cost vs all-Opus / all-Sonnet, with per-model Ollama tokens and estimated savings.

🔬Real token accounting

Beyond estimates: get_real_token_usage parses Claude Code transcripts for true per-tier tokens (incl. cache pricing) — main session and every sub-agent.

🧮170k context budget

Planner & Reviewer carry a 170k per-subtask context budget, verified post-run against real transcript counts and flagged if exceeded.

Flags

Drop any of these anywhere in the task text.

FlagEffect
[auto]Skip the high-risk plan confirmation halt (CI / unattended runs).
[model:<name>]Pin the Worker-generation model (Ollama tag or OpenRouter id); skips the auto-probe and scoring.
[openrouter]Use OpenRouter as the Worker-generation backend instead of Ollama (separate path). Needs OPENROUTER_API_KEY + OPENROUTER_MODEL; falls back to Haiku if unconfigured.
[ollama-only]Ollama writes the code; Haiku writes it to the file verbatim (no adaptation).
[ollama-agent]The Ollama model drives the whole step via its own tool-calling loop — no Haiku Worker. For tool-call-capable models; falls back to Haiku if it can't.
[fast-select]Skip llm-checker scoring; just use the first installed model.

Install & use

Requires Claude Code, Python 3.10+, and (optionally) Ollama for local offload.

Install

git clone https://github.com/Adam-Luciano-MDB/\
multi-model-flow.git
cd multi-model-flow
./scripts/setup_mcp.sh --global
# registers the plugin as a marketplace,
# installs it, wires the Ollama MCP server.
# then restart Claude Code

Use from any project

/mmf Build a CSV parser with unit tests.

/mmf [auto] Add rate limiting to /api/v2.

/mmf [ollama-agent] [model:qwen2.5-coder:7b]
   Add a /health endpoint with a test.

After a run, the cost dashboard opens at http://localhost:8765, or ask Claude: "Use the ollama-local get_real_token_usage tool."

Bundled MCP tools

The ollama-local server ships with the plugin — these are callable from the skill or directly from Claude.

  • list_local_models — installed Ollama models
  • list_models_for_selection — numbered pick list
  • recommend_model — RAM-based recommender
  • get_model_context_length — model's max window
  • estimate_context_fit — does a step fit the window?
  • ask_local_model — raw local generation
  • ask_local_model_for_code — code-optimised generation
  • run_ollama_coding_agent — local tool-calling loop
  • check_token_budget — 170k budget audit
  • get_real_token_usage — real per-tier tokens & cost
  • get_metrics_summary — CLI metrics summary
  • open_metrics_dashboard — launch the web dashboard
  • log_event — append a metrics record
